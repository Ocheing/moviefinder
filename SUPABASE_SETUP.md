# Supabase Setup Guide for MovieFinder

To make your MovieFinder project full dynamic and realtime with Supabase, you need to set up your project and database columns. 

## 1. Project Creation
- Go to [Supabase](https://supabase.com) and create a new project.
- In your project dashboard, go to **Project Settings > API** to find your **Project URL** and **anon public key**.
- Paste those into `script.js` at the top where `SUPABASE_URL` and `SUPABASE_ANON_KEY` are currently "YOUR_SUPABASE_PROJECT_URL" and "YOUR_SUPABASE_ANON_KEY".

## 2. Table Creation (SQL Editor)
Go to the **SQL Editor** in your Supabase dashboard and run the following commands to create your database tables and enable Realtime row-level features.

```sql
-- 1. Create the 'favorites' table
CREATE TABLE public.favorites (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  movie_id INT NOT NULL,
  movie_data JSONB NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Ensure a user can't favorite the same movie twice
ALTER TABLE public.favorites ADD CONSTRAINT unique_favorite UNIQUE (user_id, movie_id);

-- 2. Create the 'watchlist' table
CREATE TABLE public.watchlist (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  movie_id INT NOT NULL,
  movie_data JSONB NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Ensure a user can't watchlist the same movie twice
ALTER TABLE public.watchlist ADD CONSTRAINT unique_watchlist UNIQUE (user_id, movie_id);

-- 3. Create the 'user_ratings' table
CREATE TABLE public.user_ratings (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  movie_id INT NOT NULL,
  rating INT NOT NULL CHECK (rating >= 1 AND rating <= 5),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Ensure a user has only one active rating per movie
ALTER TABLE public.user_ratings ADD CONSTRAINT unique_rating UNIQUE (user_id, movie_id);
```

## 3. Enable Realtime Sync
To ensure instant updates whenever data changes (so the project is in "real-time"):

1. Go to **Database > Publications**.
2. For the `supabase_realtime` publication, verify that the `favorites`, `watchlist`, and `user_ratings` tables are enabled. 
*(Alternatively, you can run this SQL:)*
```sql
ALTER PUBLICATION supabase_realtime ADD TABLE favorites;
ALTER PUBLICATION supabase_realtime ADD TABLE watchlist;
ALTER PUBLICATION supabase_realtime ADD TABLE user_ratings;
```

## 4. Setting Up Row Level Security (RLS) policies
By default, Supabase secures tables. We need policies so users can only view and modify their *own* data. Run exactly this in the SQL Editor:

```sql
-- Enable RLS
ALTER TABLE public.favorites ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.watchlist ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.user_ratings ENABLE ROW LEVEL SECURITY;

-- Favorites policies
CREATE POLICY "Users can insert their own favorites" ON public.favorites FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can view their own favorites" ON public.favorites FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can delete their own favorites" ON public.favorites FOR DELETE USING (auth.uid() = user_id);

-- Watchlist policies
CREATE POLICY "Users can insert their own watchlist" ON public.watchlist FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can view their own watchlist" ON public.watchlist FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can delete their own watchlist" ON public.watchlist FOR DELETE USING (auth.uid() = user_id);

-- User_ratings policies
CREATE POLICY "Users can update their own ratings" ON public.user_ratings FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can insert their own ratings" ON public.user_ratings FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can view their own ratings" ON public.user_ratings FOR SELECT USING (auth.uid() = user_id);
```

Once this is complete and you place your `SUPABASE_URL` and `SUPABASE_ANON_KEY` inside `script.js`, your app will be securely connected and globally synced across your devices in real-time!
